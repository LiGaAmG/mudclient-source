# mudclient-source (fork LiGaAmG)

Клиент для текстовых RPG/MUD-игр. Форк [syrompetka/mudclient](https://github.com/syrompetka/mudclient) с геймплейными фичами и оптимизациями производительности.

## Маршруты (Route system)

- **Автогенерация маршрутов** между всеми зонами — BFS без привязки к хабу, кнопка в диалоге (167 зон, ~307 маршрутов)
- **Умный переход по подстроке** — `маршрут идти <часть названия>`
- Поиск в диалогах «Идти в» и «Удалить маршрут»
- **Лукахед (pipeline)** — `маршрут лукахед N` отправляет N ходов вперёд не дожидаясь подтверждения каждой комнаты, ускоряет прохождение длинных маршрутов. `маршрут лукахед 0` — выключить. Настройка сохраняется между сессиями. Применяется только к явным пользовательским командам — травник всегда двигается синхронно
- `маршрут справка` — список всех команд

## Травник (Herb Gatherer)

Отдельная подсистема автосбора трав:

- Кросс-зонный обход травяных клеток с Dijkstra-оптимизацией маршрута между зонами
- Автовключение невидимости, автодетект навыка сбора
- База 18 трав, разметка клеток прямо на карте
- Watchdog/retry-таймеры на случай зависаний, автоматический возврат домой после завершения обхода
- Команды: `травник старт` / `травник старт опасно` / `травник старт очень опасно`, `травник стоп`, `травник домой`, `травник справка`

## Менеджер заклинаний (SpellManager)

Отдельный плагин (~2100 строк) для учёта и автоматизации заучивания заклинаний, парсит вывод сервера в реальном времени:

- Парсинг ответов на `зауч` и `закл` — заученные, в процессе запоминания, свободные слоты по кругам
- **План заучивания** — для каждого заклинания задаётся желаемое количество (`Desired`); команда `мем план` рассылает `зауч !<заклинание>` под доступные слоты, пока план не выполнен
- **Перезаучивание** — команда `мем` сравнивает текущий набор с планом и сама шлёт `зауч стоп`/`зауч забыть` на лишнее и доучивает недостающее
- Отдельные виджеты: полный менеджер заклинаний на вкладку, компактный счётчик, и **глобальный счётчик** — агрегирует заклинания сразу по всем открытым вкладкам (удобно при игре сразу несколькими персонажами)
- План сохраняется на диск на вкладку (по стабильному имени таба, переживает переоткрытие)
- Батчинг строк на диспетчере (не более одного `BeginInvoke` в очереди) + быстрый pre-filter по префиксам строк — не молотит regex на каждой строке чата

## Monster ID / таргетинг

- `$monsteridN` — целеуказание по индексу с отображением id1/id2 голубым цветом в GroupWidget
- Умный матчинг при обновлении списка монстров
- `$monster1` корректно работает для монстров с дефисом в имени («Пигмей-маг»)
- `$monsterLast` — последний монстр в комнате

## Lore (StuffDatabase)

- Автохайлайт известных вещей прямо в игровом тексте
- **Drop-locations** — автозапись «с какого монстра в какой зоне» при подборе предмета, тултип «Падает с»
- Команды `лорд` / `лорд удалить N`, `лорк` (удаляет комментарий без аргумента), `лор справка`
- Улучшенный парсинг `[[ItemName]]`, матчинг количества/качества

## Эффекты (Affects)

- Добавлены 23 новых эффекта: Затуманивание разума, Затуманивание, Слово тьмы, Окостенение, Групповая точность, Успокоение, Освящение, Магический щит, Ледяной щит, Духовная аура, Защита от света, Защита от тьмы, Каменная кожа, Живое прикосновение, Увеличение, Медитация, Дыхание под водой, Боль, Портал, Чувствовать жизнь, Инфракрасное зрение, Видеть невидимое, Видеть магию
- Иконки в GroupWidget для Portal / SenseLife / Infravision / SeeInvisible / SeeMagic
- Матчинг без учёта регистра и `_`/пробел; новые эффекты подхватываются в настройках автоматически

## Lua-скриптинг

Полноценная песочница для написания игровой логики на Lua (NLua):

- Instruction-count watchdog защищает от зависаний в бесконечных циклах
- Корутин-скедулер: `StartScript` / `StopScript` / `Wait(ms)` / `WaitGroupState` / `WaitRoomState` / `WaitRoomChange`
- Полный доступ к полям `CharacterStatus`/`MonsterStatus`, `group[]`, локальным данным зоны/комнаты
- API: `SetVariable` / `GetVariable` / `Echo` / `EnableGroup` / `DisableGroup` / `SetStatus` / `SendToWindow` / `SendToAllWindows` / `SendCommand`
- `LuaScriptAction` — новый тип действия в триггерах и алиасах
- Редактор с подсветкой синтаксиса, диалог Scripts с live-статусом и кнопками Start/Stop
- Скрипты сохраняются на профиль (`Scripts.xml`), горячая перезагрузка в открытые вкладки при сохранении
- Отдельный раздел в Help с категоризированными темами и примерами кода

## Производительность

Отдельная многомесячная работа над лагами клиента:

- **Удалён SOCKS5-прокси** — оказался источником сетевых лагов, из фичи стал техдолгом и был выпилен
- **Триггеры и замены**: индексный диспетчинг вместо линейного перебора, any-of pre-check, fast-path pre-check, regex required-literal pre-check, кэширование var-regex
- **Лор**: line-level кэш известных вещей, устранена лишняя работа в `MarkKnownLoreItems`, устранены конвейерные лаги при флуде текста от сервера
- **Рендеринг**: пул viewmodel-строк для виджета монстров вместо пересоздания, отключены холостые обновления скрытых виджетов, VisualBrush-иконки вместо растеризации в bitmap на старте, WPF storyboard-анимации вместо ручного мигания (общий таймер BlinkPulse), вложенный BitmapCache, устранены лишние перепривязки DataGrid при переключении вкладок
- **Сеть**: алгоритм Нейгла отключён (`TCP NoDelay`), синхронная построчная загрузка зон на границах заменена фоновым прогревом всех зон при старте
- **Маршруты**: `RebuildRouteIndexes` ускорен с O(N³) до O(N) через словарь смежности вместо `Except()` на каждом шаге рекурсии; загрузка `Routes.xml` вынесена в фоновый поток — клиент стартует мгновенно вместо зависания на секунды
- **StuffDB**: кэш, батчинг рендера, per-tab RTT-индикатор, throttle GroupWidget
- Встроенная диагностика: команда `#perf`, perf-лог с маячками (`UI_OP`, `SEND/NET`, `ROOM*`, `TIMER_FIRE`, `SEND_LAG`), автозонд медленных команд
